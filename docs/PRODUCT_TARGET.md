# Product Target

Admin Prototype Studio is now a Canvas-first Backoffice Prototype Editor.

The product should let users arrange pages like Axure, configure real backoffice components from Ant Design and ProComponents, preview interactive admin prototypes, and export PRD documentation in plain business language.

Core principle:

> Freeform operation, automatically maintained structure.

Current phase principle:

> Close the editing workflow: operate on the canvas, organize layers, reuse assets, configure basic interactions, and preview the result.

Users should be able to drag, move, resize, align, copy, group, right-click, and double-click-edit objects on an infinite canvas. The underlying project model still preserves component type, props, content, data, events, canvas placement, and business semantics as serializable project data.

The product remains frontend-only and MVP-scoped. It is not a production low-code platform, deployment generator, database builder, collaboration product, or enterprise RBAC system.

The current S4 focus is workflow continuity: space-pan and canvas shortcuts, layer management, template save/reuse, recent assets, runtime visibility/interactivity actions, and PRD output that describes these behaviors in business language.
